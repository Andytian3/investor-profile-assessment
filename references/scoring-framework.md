## 判分方法

先看案例卡，再用开放题修正。

1. `standard` 模式先跑 `8-10` 张核心案例卡。
2. 不要把单题当结论，要看同类信号是否反复出现。
3. 用开放题确认真实资金约束、最大回撤经历、目标收益和持有周期。
4. 如果案例卡答案和真实经历冲突，优先参考真实经历。
5. 如果用户回答明显带有“想成为怎样的人”，但历史行为不是那样，按历史行为约束。

## 评分原则

- 每个维度优先打 1-5 分，必要时允许 `unknown`。
- 只根据证据打分，不根据用户自我标签直接打分。
- 一次访谈得到的是“当前画像”，不是永久结论。
- 硬约束优先于偏好，历史行为优先于空泛表态。

## 校准与降档规则

- `历史上扛过多大回撤` 不等于 `未来愿意再承受多大回撤`。前者主要用于判断 `Risk Tolerance`，后者必须进入 `hard constraints`。
- 如果用户在大回撤中的主要行为是“被动躺平等待”，而不是复盘和修正，不要仅凭“没卖出”就把 `Behavioral Discipline` 打到 `4-5`。
- 如果证据横跨相邻两个分档，默认先取较低档；只有当开放题明确说明主要资金和主要持有行为落在更高档时，才上调。
- `Horizon Commitment` 出现“`1-3` 年，甚至更久”这类混合表述时，默认先按 `3` 分处理；只有当核心资本确实可放 `3` 年以上时，再评为 `4`。
- 如果用户过去经历过极端回撤，但明确表示未来不接受同等级别回撤，最终建议和约束要按“未来接受上限”落地，而不是按历史极值外推。

| --- | --- | --- |

| `C2` | `Risk Tolerance`, `Horizon Commitment` | 判断回撤接受度与高信念程度 |
| `C3` | `Risk Tolerance`, `Behavioral Discipline` | 判断组合级别波动时是否稳定 |
| `C4` | `Behavioral Discipline` | 判断 FOMO 与社交噪音敏感度 |
| `C5` | 风格标签 | 判断是否偏长期质量与基本面 |
| `C6` | 风格标签, `Horizon Commitment` | 判断是否偏价值修复与耐心 |
| `C7` | 风格标签, `Return Ambition` | 判断是否愿为高质量成长付费 |
| `C8` | `Concentration Preference` | 判断集中度偏好 |
| `C9` | `Process Structure` | 判断是否有明确研究与复盘框架 |
| `C10` | `Process Structure`, `Behavioral Discipline` | 判断能否修正错误与避免锚定 |
| `C11` | `Complexity Appetite` | 判断是否适合高维护策略，并辅助维护负荷匹配 |

| `C14` | `Horizon Commitment`, `Risk Tolerance` | 判断长期兑现耐心 |

实际打分时，用下面的经验规则：

- 同一维度如果大部分信号都偏保守，通常落在 `1-2` 分。
- 如果信号混合，或案例卡和开放题互相抵消，通常落在 `3` 分。
- 如果同类高强度信号反复出现，且开放题也支持，通常落在 `4-5` 分。
- 单个“勇敢”的答案不能直接把风险等级拉高，必须同时看 `Risk Capacity` 和 `Behavioral Discipline`。

## 核心维度

### 1. Risk Capacity

衡量用户在财务上客观能承受多大波动。

| 分数 | 判定锚点 |
| --- | --- |
| 1 | 资金一年内要用，亏损会影响生活或关键计划，没有清晰缓冲垫 |
| 2 | 有一定缓冲，但提款概率较高，现金流或收入不稳定 |
| 3 | 资金相对独立，1-3 年可承受波动，但仍有明确流动性要求 |
| 4 | 有长期资本池，生活不受组合波动直接影响 |
| 5 | 资金明确长期化，回撤不影响生活与主要规划，有充分缓冲 |

### 2. Risk Tolerance

衡量用户心理上能承受多大回撤和波动。

| 分数 | 判定锚点 |
| --- | --- |
| 1 | 5%-10% 的亏损就明显焦虑或倾向止损离场 |
| 2 | 10%-15% 的亏损开始明显不适，容易频繁调整 |
| 3 | 能接受 15%-25% 的回撤，但需要较强解释与陪伴 |
| 4 | 能接受 25%-40% 的波动，只要逻辑未破坏 |
| 5 | 能接受 40% 以上高波动，并愿意基于假设变化而非情绪做决定 |

### 3. Return Ambition

衡量用户对收益的追求强度。

| 分数 | 判定锚点 |
| --- | --- |
| 1 | 以保本、跑赢现金或通胀为主 |
| 2 | 以稳健增值为主，愿意牺牲上行换稳定 |
| 3 | 接受中等波动换取长期复利 |
| 4 | 明确追求显著跑赢基准或高成长收益 |
| 5 | 主动寻找高弹性、高赔率机会，接受较大失败率 |

### 4. Horizon Commitment

衡量用户能否给策略留出兑现时间。

| 分数 | 判定锚点 |
| --- | --- |
| 1 | 少于 6 个月，结果导向很强 |
| 2 | 6-12 个月，耐心有限 |
| 3 | 1-3 年 |
| 4 | 3-5 年 |
| 5 | 5 年以上，且能接受中间波动 |

### 5. Liquidity Flexibility

衡量用户是否能接受较低流动性或较慢兑现节奏。

| 分数 | 判定锚点 |
| --- | --- |
| 1 | 强流动性要求，可能随时提款 |
| 2 | 流动性需求较高，不适合锁仓或慢兑现 |
| 3 | 中性，能接受部分资金中期占用 |
| 4 | 大部分资金可接受低频调整 |
| 5 | 明确能接受长期锁定或低流动性配置 |

### 6. Concentration Preference

衡量用户更偏分散还是高信念集中。

| 分数 | 判定锚点 |
| --- | --- |
| 1 | 明显偏好多元分散，避免单一仓位风险 |
| 2 | 以分散为主，单一仓位上限较低 |
| 3 | 接受核心-卫星或中等集中 |
| 4 | 愿意对高信念机会给较大权重 |
| 5 | 明确接受少数重仓甚至单点高信念 |

### 7. Process Structure

衡量用户决策过程是否结构化。

| 分数 | 判定锚点 |
| --- | --- |
| 1 | 主要依赖感觉、消息面或他人观点 |
| 2 | 有零散方法，但缺少一致框架 |
| 3 | 有基本清单，会看关键证据 |
| 4 | 有相对稳定的研究和复盘流程 |
| 5 | 过程高度结构化，买入、加减仓、失效条件都清晰 |

### 8. Behavioral Discipline

衡量用户在压力下是否能稳定执行。

| 分数 | 判定锚点 |
| --- | --- |
| 1 | 容易 panic sell、FOMO、追涨杀跌 |
| 2 | 情绪波动会明显影响仓位与判断 |
| 3 | 偶有情绪化，但能部分纠偏 |
| 4 | 大多能按预设逻辑执行 |
| 5 | 高压环境下仍能稳定、复盘并修正 |

### 9. Complexity Appetite

衡量用户是否适合复杂工具或复杂策略。

| 分数 | 判定锚点 |
| --- | --- |
| 1 | 只适合简单、透明、低维护方案 |
| 2 | 能接受少量复杂度，但不适合衍生结构 |
| 3 | 可接受中等复杂度和规则化策略 |
| 4 | 愿意使用更复杂工具，前提是风险可解释 |
| 5 | 主动接受复杂结构、衍生品或多层策略，并理解风险路径 |

## 风险总分计算

先计算基础风险分：

`base_risk = 0.24*Risk Capacity + 0.24*Risk Tolerance + 0.14*Return Ambition + 0.14*Horizon Commitment + 0.12*Liquidity Flexibility + 0.12*Concentration Preference`

再做修正：

- 如果 `Behavioral Discipline <= 2` 且 `base_risk >= 4.0`，下调 0.5 分。
- 如果 `Risk Capacity <= 2` 且 `Risk Tolerance >= 4`，最终风险等级最高不超过 3。
- 如果 `Process Structure <= 2` 且 `Complexity Appetite >= 4`，保留原风险等级，但添加 `execution-risk-warning`。
- 如果 `Liquidity Flexibility <= 2`，未来分析默认不推荐低流动性策略，除非用户明确要求。
- 如果 `历史最大回撤` 明显高于 `未来可接受回撤`，保留历史经历用于画像，但额外添加 `drawdown-gap-warning`。

最终输出时，最好显式给出：

- `base_risk_score`
- `final_risk_score`
- 所有触发过的修正与警报

最终风险等级分层：

| 区间 | 风险标签 |
| --- | --- |
| 1.0-1.8 | 资本保全型 |
| 1.81-2.6 | 稳健收益型 |
| 2.61-3.4 | 均衡成长型 |
| 3.41-4.2 | 进取增长型 |
| 4.21-5.0 | 高波动机会型 |

## 风格标签判定

除了风险标签，再给出 1 个主风格和 1 个副风格。优先从访谈证据而不是主观自称来推断。

可用风格标签：

- `income-defensive`：重视现金流、稳定性、分红、回撤控制
- `balanced-allocator`：偏资产配置、分散、再平衡、稳中求进
- `fundamental-conviction`：重研究、重逻辑、愿意长期持有高信念标的
- `growth-compounder`：偏优质成长、长期复利、容忍估值波动
- `value-reversion`：偏低估值、安全边际、均值回归
- `trend-opportunist`：偏价格趋势、强弱判断、节奏交易
- `event-driven`：偏事件催化、时点博弈、赔率机会
- `exploratory-high-beta`：偏主题、创新、高波动、高弹性资产

判定提示：

- `C6=yes` 且 `C7=no` 时，更接近 `value-reversion`。
- `C5=yes`、`C7=yes`、`C14=yes` 时，更接近 `growth-compounder`。
- `C5=yes`、`C8=yes`、`C14=yes` 时，更接近 `fundamental-conviction`。
- `C13=yes` 且 `C8=no` 时，更接近 `income-defensive` 或 `balanced-allocator`。
- `C4=yes`、`C11=yes`，且更在意节奏与机会时，更接近 `trend-opportunist`。
- 对主题、高波动和高弹性机会明显更兴奋时，更接近 `exploratory-high-beta`。

## 约束与警报规则

未来个性化分析必须继承这些规则：

- `hard constraints`：由资金用途、流动性需求、最大可承受回撤、复杂度上限构成。
- `soft preferences`：由风格、持有周期、研究方式、表达偏好构成。
- 如果用户历史行为与口头偏好冲突，默认以历史行为约束，并明确写出冲突。
- 如果用户高 ambition 但低 discipline，不要直接给更激进建议，先给更强的仓位和复盘机制。
- 如果用户在 `C12` 上回答 `是`，但又无法清楚解释最坏损失路径，默认添加 `execution-risk-warning`。
- 如果多个维度证据不足，降低画像置信度，不强行生成非常具体的策略偏好。
- 如果用户能接受的维护强度明显低于某策略要求，默认把该策略视为不匹配，即使收益预期更高。
